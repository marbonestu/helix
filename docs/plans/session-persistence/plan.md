# Session Persistence Implementation Plan

Save and restore editor sessions so that reopening Helix in a project directory
resumes with the same open buffers, view layout, cursor positions, and related
state.

## Key Insight

Helix already stores all the necessary state in well-structured types:
`Editor.documents` (BTreeMap), `Editor.tree` (split layout), `View.jumps`
(jump list), `Document.selections` (per-view cursors), and `Editor.registers`.
The task is to serialize a snapshot of this state on exit and reconstruct it on
startup. The session file lives alongside the existing workspace config in
`.helix/` or in the user cache directory.

## Architecture Overview

```
                  ┌─────────────────┐
                  │  Session File   │
                  │ (.helix/session │
                  │  or cache dir)  │
                  └───────┬─────────┘
                          │
              ┌───────────┴───────────┐
              │                       │
        save_session()          restore_session()
        (on :wq / :q)          (on startup)
              │                       │
    ┌─────────┴─────────┐   ┌────────┴────────┐
    │ SessionSnapshot   │   │ SessionSnapshot  │
    │ {documents, tree, │   │ (deserialized)   │
    │  registers, cwd}  │   │                  │
    └───────────────────┘   └──────────────────┘
```

## State Inventory

| State                  | Where it lives                          | Persist? | Notes                                      |
|------------------------|-----------------------------------------|----------|---------------------------------------------|
| Open file paths        | `Document.path`                         | Yes      | Core of session restore                     |
| View/split layout      | `Editor.tree` (Tree of View/Container)  | Yes      | Recreate split arrangement                  |
| Cursor positions       | `Document.selections[view_id]`          | Yes      | Per-view primary selection                  |
| Scroll positions       | `ViewData.view_position`                | Yes      | Avoid jarring jumps on restore              |
| Focused view           | `Tree.focus`                            | Yes      | Restore active pane                         |
| Jump list              | `View.jumps`                            | Yes      | Navigation continuity                       |
| Named registers        | `Editor.registers`                      | Phase 2  | User clipboard, macros                      |
| Search history         | Register `/`                            | Phase 2  | Part of registers                           |
| Command history        | `Prompt` history                        | Phase 3  | Stored in prompt widget                     |
| Undo history           | `Document.history`                      | No       | Complex, large, low ROI                     |
| LSP state              | `Editor.language_servers`               | No       | Reconnects automatically                    |
| Diagnostics            | `Editor.diagnostics`                    | No       | Regenerated by LSP                          |
| Breakpoints            | `Editor.breakpoints`                    | Phase 3  | Useful for debug workflows                  |
| Working directory      | `std::env::current_dir()`               | Yes      | Anchor for relative paths                   |

## Phase 1: MVP — Buffers and Layout

Minimal implementation: persist which files are open, the split layout, cursor
positions, and scroll offsets. Restore them on startup.

### Step 1.1: Define serializable session types

**File:** `helix-view/src/session.rs` (new)

```rust
use serde::{Deserialize, Serialize};
use std::path::PathBuf;

#[derive(Debug, Serialize, Deserialize)]
pub struct SessionSnapshot {
    /// Schema version for forward compatibility
    pub version: u32,
    /// Working directory at time of save
    pub working_directory: PathBuf,
    /// Layout tree describing splits and documents
    pub layout: SessionLayout,
    /// Index of the focused leaf in depth-first order
    pub focused_index: usize,
}

#[derive(Debug, Serialize, Deserialize)]
pub enum SessionLayout {
    /// A leaf view showing a document
    View(SessionView),
    /// A container splitting space among children
    Container {
        layout: SessionSplitDirection,
        children: Vec<SessionLayout>,
    },
}

#[derive(Debug, Serialize, Deserialize)]
pub enum SessionSplitDirection {
    Horizontal,
    Vertical,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct SessionView {
    /// Document shown in this view
    pub document: SessionDocument,
    /// Documents previously visited in this view (for buffer switching)
    pub docs_access_history: Vec<PathBuf>,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct SessionDocument {
    /// Absolute path to the file (None for scratch buffers)
    pub path: Option<PathBuf>,
    /// Primary cursor position as (anchor_char_offset, head_char_offset)
    pub selection: Option<(usize, usize)>,
    /// Scroll position as (anchor_line, horizontal_offset)
    pub view_position: Option<(usize, usize)>,
}
```

### Step 1.2: Implement snapshot creation from Editor state

**File:** `helix-view/src/session.rs`

```rust
impl SessionSnapshot {
    pub fn capture(editor: &Editor) -> Self {
        let working_directory = std::env::current_dir()
            .unwrap_or_default();

        let (layout, focused_index) = capture_tree(&editor.tree, &editor.documents);

        SessionSnapshot {
            version: 1,
            working_directory,
            layout,
            focused_index,
        }
    }
}

fn capture_tree(
    tree: &Tree,
    documents: &BTreeMap<DocumentId, Document>,
) -> (SessionLayout, usize) {
    // Walk the Tree structure depth-first:
    // - Container nodes → SessionLayout::Container with recursive children
    // - View nodes → SessionLayout::View with document path and selection
    // Track which leaf index corresponds to tree.focus
    // ...
}

fn capture_view(
    view: &View,
    documents: &BTreeMap<DocumentId, Document>,
) -> SessionView {
    let doc = &documents[&view.doc];
    let selection = doc.selection(view.id);
    let primary = selection.primary();
    let view_position = doc.view_data(view.id)
        .map(|vd| (vd.view_position.anchor, vd.view_position.horizontal_offset));

    SessionView {
        document: SessionDocument {
            path: doc.path().map(|p| p.to_path_buf()),
            selection: Some((primary.anchor, primary.head)),
            view_position,
        },
        docs_access_history: view.docs_access_history
            .iter()
            .filter_map(|id| documents.get(id)?.path().map(|p| p.to_path_buf()))
            .collect(),
    }
}
```

### Step 1.3: Implement session file I/O

**File:** `helix-view/src/session.rs`

```rust
use std::fs;
use std::io;

const SESSION_FILENAME: &str = "session.json";

/// Resolve the session file path.
/// Prefer workspace `.helix/session.json` if `.helix/` exists,
/// otherwise fall back to `~/.cache/helix/sessions/<dir_hash>.json`.
pub fn session_path() -> PathBuf {
    let workspace = helix_loader::workspace_config_dir();
    if workspace.exists() {
        return workspace.join(SESSION_FILENAME);
    }
    let cache = helix_loader::cache_dir();
    let cwd = std::env::current_dir().unwrap_or_default();
    let hash = hash_path(&cwd);
    cache.join("sessions").join(format!("{hash}.json"))
}

pub fn save_session(snapshot: &SessionSnapshot) -> io::Result<()> {
    let path = session_path();
    if let Some(parent) = path.parent() {
        fs::create_dir_all(parent)?;
    }
    let json = serde_json::to_string_pretty(snapshot)?;
    fs::write(&path, json)?;
    Ok(())
}

pub fn load_session() -> io::Result<SessionSnapshot> {
    let path = session_path();
    let json = fs::read_to_string(&path)?;
    let snapshot: SessionSnapshot = serde_json::from_str(&json)?;
    Ok(snapshot)
}

pub fn delete_session() -> io::Result<()> {
    let path = session_path();
    if path.exists() {
        fs::remove_file(&path)?;
    }
    Ok(())
}

fn hash_path(path: &Path) -> String {
    use std::hash::{Hash, Hasher};
    let mut hasher = std::collections::hash_map::DefaultHasher::new();
    path.hash(&mut hasher);
    format!("{:x}", hasher.finish())
}
```

### Step 1.4: Save session on exit

**File:** `helix-term/src/application.rs`

In `Application::close()`, before shutting down language servers:

```rust
async fn close(&mut self) -> anyhow::Result<()> {
    // ... existing job finish and flush_writes ...

    // Save session if configured
    if self.editor.config().session.persist {
        let snapshot = SessionSnapshot::capture(&self.editor);
        if let Err(e) = session::save_session(&snapshot) {
            log::warn!("Failed to save session: {e}");
        }

    // ... existing close_language_servers ...
```

### Step 1.5: Restore session on startup

**File:** `helix-term/src/application.rs`

In `Application::new()`, after editor creation but before opening CLI-specified
files:

```rust
// If no files specified on CLI and session restore is enabled, try restoring
if args.files.is_empty() && config.editor.session.persist {
    match session::load_session() {
        Ok(snapshot) => {
            if let Err(e) = restore_session(&mut editor, snapshot) {
                log::warn!("Failed to restore session: {e}");
                // Fall through to default empty buffer
            }
        }
        Err(_) => {
            // No session file or parse error — start fresh
        }
    }
}
```

Restoration logic:

```rust
fn restore_session(editor: &mut Editor, snapshot: SessionSnapshot) -> anyhow::Result<()> {
    // 1. Walk SessionLayout tree
    // 2. For each SessionView leaf:
    //    a. Open the document via editor.open() if path exists on disk
    //    b. Skip files that no longer exist (log warning)
    //    c. Restore cursor position (clamped to document length)
    //    d. Restore scroll position
    // 3. Recreate split layout by calling editor.tree.split() to match
    //    the saved Container structure
    // 4. Set focus to saved focused_index
    Ok(())
}
```

### Step 1.6: Add session configuration

**File:** `helix-view/src/editor.rs`

Add to the `Config` struct:

```rust

```

```rust
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "kebab-case", default, deny_unknown_fields)]
pub struct SessionConfig {
    /// Enable automatic session save/restore
    pub persist: bool,
    /// What to persist: "layout" (default), "full"
    pub scope: SessionScope,
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "kebab-case")]
pub enum SessionScope {
    /// Buffers, layout, cursors, scroll positions
    Layout,
    /// Layout + registers, search history
    Full,
}

impl Default for SessionConfig {
    fn default() -> Self {
        Self {
            persist: false,
            scope: SessionScope::Layout,
        }
    }
}
```

Config usage in `config.toml`:

```toml
[editor.session]
persist = true
scope = "layout"  # or "full"
```

### Step 1.7: Add session commands

**File:** `helix-term/src/commands/typed.rs`

```rust
fn session_save(cx: &mut compositor::Context, _args: &[Cow<str>], event: PromptEvent) {
    let snapshot = SessionSnapshot::capture(cx.editor);
    match session::save_session(&snapshot) {
        Ok(()) => cx.editor.set_status("Session saved"),
        Err(e) => cx.editor.set_error(format!("Failed to save session: {e}")),
    }
}

fn session_restore(cx: &mut compositor::Context, _args: &[Cow<str>], event: PromptEvent) {
    match session::load_session() {
        Ok(snapshot) => {
            match restore_session(cx.editor, snapshot) {
                Ok(()) => cx.editor.set_status("Session restored"),
                Err(e) => cx.editor.set_error(format!("Failed to restore: {e}")),
            }
        }
        Err(e) => cx.editor.set_error(format!("No session found: {e}")),
    }
}

fn session_delete(cx: &mut compositor::Context, _args: &[Cow<str>], event: PromptEvent) {
    match session::delete_session() {
        Ok(()) => cx.editor.set_status("Session deleted"),
        Err(e) => cx.editor.set_error(format!("Failed to delete session: {e}")),
    }
}
```

Register in `TYPABLE_COMMAND_LIST`:

```rust
}
```

### Step 2.2: Capture and restore registers

During `SessionSnapshot::capture()`, if scope is `Full`:

```rust
if scope == SessionScope::Full {
    snapshot.registers = Some(capture_registers(&editor.registers));
}
```

Only persist user-writable named registers (`a-z`, `"`, `/`). Skip read-only
special registers (`#`, `.`, `%`) and clipboard registers (`*`, `+`) which
sync with the system.

During restore, merge saved register values back into `Editor.registers`:

```rust
fn restore_registers(registers: &mut Registers, saved: HashMap<char, Vec<String>>) {
    for (name, values) in saved {
        // Skip special/read-only registers
        if matches!(name, '_' | '#' | '.' | '%' | '*' | '+') {
            continue;
        }
        for value in values {
            registers.push(name, value);
        }
    }
}
```

## Phase 5: Edge Cases and Polish

### File no longer exists
Skip the document during restore, log a warning, and show a status message
listing skipped files after restore completes.

### File modified externally
Helix already detects external modifications and prompts for reload. The
restored cursor position is clamped to the current document length to prevent
panics.

### Scratch buffers
Scratch buffers (no path) cannot be persisted in Phase 1. Phase 3 could
optionally serialize their content inline for small buffers.

### Schema migration
The `version` field in `SessionSnapshot` enables forward compatibility. When
loading an older version, apply migration logic. Unknown fields are ignored via
`#[serde(default)]`.

### Multiple Helix instances
If two instances share the same project directory, the last one to exit wins.
This matches Vim/Neovim behavior. A future enhancement could use file locking
or instance-specific session names.

### Session file location precedence
1. Workspace `.helix/session.json` — if `.helix/` directory exists
2. `~/.cache/helix/sessions/<cwd_hash>.json` — fallback for projects without `.helix/`

### Performance
Session files are small (typically <10KB even with hundreds of buffers).
JSON serialization/deserialization adds negligible startup time. Use
`serde_json` which is already a workspace dependency.
